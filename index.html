CybersécuritéDéveloppement

Transmission sécurisée de données via MQTT

Transmission de données environnementales simulées entre un équipement et une entreprise via un broker MQTT, avec un chiffrement bout-en-bout (AES-CBC). Le broker ne voit jamais les messages en clair.

Détails techniques

Contexte : SAE de développement (école d'ingénieur), architecture à 3 VM Ubuntu (équipement, entreprise, broker Mosquitto).

Mise en œuvre : clé AES dérivée d'un identifiant room via PBKDF2-HMAC-SHA256 ; topic MQTT dérivé par SHA-256 ; configuration (IP du broker) externalisée dans un fichier dédié pour éviter le codage en dur.

Difficultés : dérivation de clé correcte, gestion du topic sécurisé, séparation des responsabilités (crypto / réseau / configuration).

Résultat : chaîne complète fonctionnelle — message chiffré côté équipement, déchiffré côté entreprise, illisible au niveau du broker.

Améliorations possibles : sel aléatoire par message, vérification d'intégrité (HMAC), stockage côté entreprise.

Réalisé avec Mahé Begnis.

CybersécuritéSystèmes & réseaux

Supervision d'une infrastructure avec Grafana

Stack de monitoring temps réel (métriques système et données métier) déployée avec Docker, dans le cadre du module « Supervision de la sécurité ». Environnement pédagogique fictif (bases polaires TAAF).

Détails techniques

Architecture : Prometheus (métriques), Grafana (dashboards), Loki (logs), Alloy (collecte), node_exporter et postgres_exporter, bases PostgreSQL — le tout orchestré via Docker Compose.

Travail réalisé : dashboards de supervision des conteneurs et des données métier, requêtes PromQL et SQL, règles d'alertes (base inaccessible, mémoire, espace disque), analyse des performances I/O disque et corrélation des logs via Loki.

Difficultés : configuration cohérente de services multiples, écriture de requêtes PromQL pertinentes, mise en place d'un alerting utile.

Résultat : rapport de TP de 58 pages documentant la démarche, les requêtes et les interprétations.

Améliorations possibles : suivi des redémarrages de conteneurs, alertes mémoire proactives.

Data / IA

Classification de la potabilité de l'eau

Modèle de classification binaire (eau potable / non potable) par régression logistique sur un jeu de données Kaggle « Water Quality » (~6 400 échantillons exploitables), avec une analyse exploratoire complète.

Détails techniques

Pipeline : nettoyage des données (valeurs #NUM! → NaN, gestion des manquants), analyse exploratoire (corrélations, boxplots / outliers, pairplot), création de variables de synthèse (ex. somme des métaux lourds), standardisation, puis analyse de la multicolinéarité par VIF.

Modélisation : régression logistique avec lecture des coefficients et des p-values via statsmodels pour interpréter la significativité de chaque contaminant.

Résultat : modèle convergent (pseudo-R² ≈ 0,30) identifiant des variables significatives (aluminium, arsenic, cadmium, perchlorate…). Le jeu de données présente un déséquilibre de classes marqué, pris en compte dans l'interprétation.

Améliorations possibles : rééquilibrage des classes, comparaison avec d'autres modèles (arbres de décision), optimisation du seuil de décision.

Réalisé avec Mahé Begnis.

Embarqué / IoT

Souris Bluetooth gestuelle sur M5Stack Core2

Transformation d'un M5Stack Core2 (ESP32) en souris Bluetooth HID pilotée par l'inclinaison de l'appareil, avec zones de clic à l'écran et retours audio embarqués.

Détails techniques

Fonctionnement : lecture de l'IMU (pitch / roll) pour déplacer le curseur, zones gauche / droite de l'écran pour les clics, bouton central pour activer le scroll, et signaux sonores (démarrage, confirmation de connexion).

Outils : développement et flash via arduino-cli, librairies M5Unified (affichage + audio) et un fork patché de la librairie BLE Mouse (T-vK).

Difficultés : librairie BLE absente de l'index Arduino (installation et patch manuels), contraintes mémoire (intégration des sons en Flash via PROGMEM), conversion des fichiers WAV en en-têtes .h, limitation à une seule connexion Bluetooth simultanée.

Résultat : souris BLE fonctionnelle (déplacement par inclinaison, clics, scroll) avec interface et retours audio.

Améliorations possibles : calibration des seuils de sensibilité, scroll horizontal, nettoyage de l'onglet de debug.

Réalisé avec Lukas Damour.

  • C++ / Arduino
  • ESP32
  • M5Unified
  • BLE HID
  • IMU

Développement logiciel

Distributeur automatique — simulation Java

Application Java en ligne de commande simulant un distributeur automatique : gestion de stock dynamique, commande multi-produits, paiement (espèces ou carte), ticket et persistance des stocks.

Détails techniques

Conception : modélisation orientée objet (produit, commande, ligne de commande, paiement, ticket, distributeur) documentée par des diagrammes UML (classes, séquence, activité) et des états de commande / modes de paiement.

Fonctionnalités : vérification stricte du stock, regroupement des produits identiques, rendu de monnaie, simulation de paiement par carte, persistance du stock dans un fichier texte entre les exécutions.

Difficultés : gestion fiable du stock pendant la commande, modélisation des différents états de paiement.

Améliorations possibles : interface graphique (Swing / JavaFX), historique des commandes, base de données relationnelle.

Projet de TP réalisé en équipe.
